# 미성이앤씨 CRM - 스킬 목록

이 디렉토리의 스킬들은 자주 사용하는 작업을 자동화합니다.

## 사용 가능한 스킬

### `/context` - 프로젝트 컨텍스트 로드
새 세션을 시작할 때 사용하세요.
- QUICK-REFERENCE.md를 읽어 프로젝트 구조 파악
- 빠른 컨텍스트 로드

**사용 예시:**
```
/context
```

### `/db` - 데이터베이스 스키마 확인
DB 테이블 구조나 쿼리 패턴을 확인할 때 사용하세요.
- DATABASE.md에서 테이블 정보 조회
- 쿼리 패턴 확인
- 테이블 관계 설명

**사용 예시:**
```
/db
/db weekly_plans 테이블 구조 알려줘
/db 목표 데이터 조회 쿼리 패턴
```

### `/feature` - 기능 확인
페이지나 API 기능을 확인할 때 사용하세요.
- FEATURES.md에서 페이지 정보 조회
- API 엔드포인트 확인
- 데이터 소스 및 로직 설명

**사용 예시:**
```
/feature
/feature 수주 실적 페이지 설명해줘
/feature API 엔드포인트 목록
```

### `/add-field` - 새 필드 추가
테이블에 새로운 컬럼을 추가할 때 사용하세요.
- SQL 마이그레이션 생성
- TypeScript 타입 업데이트
- 폼/테이블 컴포넌트 업데이트
- API 수정

**사용 예시:**
```
/add-field
/add-field weekly_plans에 memo 필드 추가
```

### `/add-stat` - 새 통계 페이지 추가
새로운 통계/분석 페이지를 추가할 때 사용하세요.
- 타입 정의
- 백엔드 API 생성
- 프론트엔드 페이지 생성
- 라우팅 및 메뉴 추가

**사용 예시:**
```
/add-stat
/add-stat 월별 수금 통계 페이지 추가
```

### `/debug-api` - API 디버깅
API에 문제가 있을 때 사용하세요.
- 로그 확인
- 디버그 로그 추가
- 일반적인 문제 해결
- 데이터 확인

**사용 예시:**
```
/debug-api
/debug-api 수주 실적 목표가 0으로 나와
/debug-api order-stats API 에러
```

---

## 스킬 사용 팁

1. **새 세션 시작**: `/context`로 프로젝트 로드
2. **기능 확인**: `/feature`나 `/db`로 필요한 정보 확인
3. **작업 수행**: `/add-field`, `/add-stat` 등으로 작업 자동화
4. **문제 해결**: `/debug-api`로 디버깅

## 스킬 vs 문서

- **스킬**: 자동화된 작업 워크플로우
- **문서** (.claude/docs/): 수동으로 읽는 참조 자료

둘 다 토큰을 절약하지만, 스킬이 더 편리합니다!
